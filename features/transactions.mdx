<AccordionGroup>
  <Accordion title="Standard Transactions">

## Transaction Status

### Status Types

<AccordionGroup>
  <Accordion title="🟡 Pending">
    Transaction submitted to network, awaiting confirmation
    
    • Average confirmation time: 10-30 seconds
    • Can be cancelled while pending
    • Fee deducted from balance
  </Accordion>

  <Accordion title="🟢 Confirmed">
    Transaction successfully processed and recorded on blockchain
    
    • Irreversible after confirmation
    • Recipient can access funds
    • Transaction hash available
  </Accordion>

  <Accordion title="🔴 Failed">
    Transaction rejected by network or failed to execute
    
    • Fees may still be deducted
    • Common reasons: insufficient balance, network issues
    • Automatic retry available for some failures
  </Accordion>

  <Accordion title="⏱️ Expired">
    Transaction not processed within timeout period
    
    • No fees deducted
    • Can be resubmitted manually
    • Usually due to network congestion
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Transaction Stuck Pending">
    **Possible Solutions:**
    
    • Wait for network congestion to clear
    • Increase priority fee for faster processing
    • Cancel and resubmit with higher fee
    • Check Solana network status
  </Accordion>

  <Accordion title="Insufficient Balance Error">
    **Check These Items:**
    
    • Account for network fees in addition to send amount
    • Verify token balance vs. SOL balance for fees
    • Consider rent-exempt balance requirements
    • Check for pending transactions
  </Accordion>

  <Accordion title="Invalid Address Error">
    **Common Causes:**
    
    • Incorrect address format or checksum
    • Mixing mainnet/devnet addresses
    • ENS domain resolution issues
    • Clipboard corruption
  </Accordion>
</AccordionGroup>
